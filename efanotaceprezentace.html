Anotace

Klíče a identita

klíče
public class Objednavka
{
    [Key]                              // PK mimo konvenci Id
    public Guid Cislo { get; set; }

    [DatabaseGenerated(DatabaseGeneratedOption.Identity)]
    public int PoradoveCislo { get; set; }

    [Timestamp]                        // rowversion
    public byte[] Verze { get; set; } = [];
}

// složený klíč – atribut NA TŘÍDĚ (EF 7+)
[PrimaryKey(nameof(StudentId), nameof(PredmetId))]
public class Zapis
{
    public int StudentId { get; set; }
    public int PredmetId { get; set; }
}
  • Konvence: Id / <Typ>Id je PK samo od sebe
  • [Key] – pro nestandardní název klíče
  • Složený PK: jen [PrimaryKey] nebo Fluent HasKeydva atributy [Key] v EF Core nefungují!
  • [DatabaseGenerated]: Identity · Computed · None
  • [Timestamp] = rowversion; [ConcurrencyCheck] = vlastní token souběžnosti
Anotace

Tabulky, sloupce a mapování

mapování
[Table("blogy", Schema = "web")]
[Comment("Firemní blogy")]
public class Blog
{
    [Column("blog_id", Order = 0)]
    public int Id { get; set; }

    [Column(TypeName = "varchar(200)")]
    public string Url { get; set; } = "";

    [NotMapped]                    // jen v paměti
    public bool MaNovinky { get; set; }
}

[Keyless] public class RadekReportu { /* view / SQL */ }
[Owned]   public class Adresa     { /* bez identity */ }
  • [Table] – název tabulky + schéma
  • [Column] – název, TypeName, pořadí sloupce
  • [Comment] – dokumentace přímo v databázi
  • [NotMapped] – vlastnost / třída bez sloupce
  • [Keyless] – pohledy a výsledky SQL (bez change trackingu)
  • [Owned] / [ComplexType] (EF 8+) – typy bez identitycomplex types v EF 10 umí i struct a JSON (přes Fluent ToJson)
Anotace

Povinnost, délky a přesnost

tvar sloupců
public class Produkt
{
    [MaxLength(80)]               // nvarchar(80)
    public string Nazev { get; set; } = "";

    [Required]                    // NOT NULL i pro string?
    public string? Poznamka { get; set; }

    [StringLength(30, MinimumLength = 3)]
    public string Kod { get; set; } = "";

    [Unicode(false)]              // varchar místo nvarchar
    [MaxLength(13)]
    public string Isbn { get; set; } = "";

    [Precision(18, 2)]            // decimal(18,2)
    public decimal Cena { get; set; }
}
  • Pozor na NRT: non-nullable string je NOT NULL už konvencí – [Required] řeší hlavně string?
  • [MaxLength] mění typ sloupce; [MinLength] a MinimumLength jsou jen validace
  • [Unicode(false)] – poloviční místo pro ASCII data (ISBN, kódy…)
  • [Precision] funguje i pro datum: [Precision(3)] → datetime2(3)
Bez [MaxLength] je string nvarchar(max) – na indexovaných sloupcích to bolí.
Anotace

Vztahy (relace)

vztahy
public class Post
{
    public int Id { get; set; }

    [ForeignKey(nameof(Blog))]        // FK mimo konvenci
    public int BlogKlic { get; set; }

    [DeleteBehavior(DeleteBehavior.Restrict)]   // EF 7+
    public Blog Blog { get; set; } = null!;
}

public class Blog
{
    public int Id { get; set; }

    [InverseProperty(nameof(Post.Blog))]
    public List<Post> Posty { get; } = [];
}
  • Konvence: BlogId se s navigací spáruje sama – atributy až mimo konvenci
  • [ForeignKey] – na FK vlastnosti, nebo na navigaci
  • [InverseProperty] – když mezi dvěma typy vede víc vztahů
  • [DeleteBehavior] – Cascade / Restrict / SetNull / NoAction…
  • [Required] na navigaci = povinný vztah (FK NOT NULL)
  • Složitější vztahy (M:N s daty navíc…) → Fluent API
Indexy

Index atributem – [Index] na třídě

stejné v .NET 9 (EF 9) i .NET 10 (EF 10)
using Microsoft.EntityFrameworkCore;   // ← [Index] je odsud!

[Index(nameof(Email), IsUnique = true)]           // unikátní
[Index(nameof(Prijmeni), nameof(Jmeno))]          // složený
[Index(nameof(Mesto), Name = "IX_Autor_Mesto")]   // vlastní název
public class Autor
{
    public int Id { get; set; }
    public string Email { get; set; } = "";
    public string Jmeno { get; set; } = "";
    public string Prijmeni { get; set; } = "";
    public string Mesto { get; set; } = "";
}
  • Atribut se dává na třídu – více atributů = více indexů
  • Složený index: na pořadí sloupců záleží (levý prefix)
  • FK sloupce dostávají index automaticky konvencí
  • Výchozí název: IX_<Tabulka>_<Sloupce>
Nezaměňovat s EF6: tam byl [Index] na vlastnosti (z …Schema). V EF Core neexistuje – jen třídní z Microsoft.EntityFrameworkCore.
.NET 9 = STS, podpora skončila 05/2026 → nové projekty rovnou na .NET 10 (LTS).
Indexy

Řazení a Fluent API

pokročilé indexy
// řazení sloupců indexu (EF 7+)
[Index(nameof(Datum), AllDescending = true)]
[Index(nameof(Stav), nameof(Datum),
       IsDescending = new[] { false, true })]
public class Objednavka { /* … */ }

// Fluent API – plná výbava (OnModelCreating)
modelBuilder.Entity<Blog>()
    .HasIndex(b => new { b.Url, b.Rating })
    .IsUnique()
    .HasDatabaseName("IX_Blog_Url_Rating")
    .HasFilter("[Url] IS NOT NULL")        // jen Fluent
    .IncludeProperties(b => b.Nazev);      // jen Fluent
  • IsDescending / AllDescending – od EF Core 7
  • Jen přes Fluent API: filtrovaný index (HasFilter) a included sloupce (IncludeProperties)
  • Více indexů nad stejnými sloupci → odlište je názvem
  • HasIndex umí vše co atribut – a víc
Praxe: běžné indexy atributem u entity (vidí je každý), speciality Fluentem v konfigurační třídě.
Pozor

Co anotacemi nejde – a co schéma nemění

Jen přes Fluent API

  • výchozí hodnoty – HasDefaultValue(Sql)
  • check constraints, sekvence
  • filtrované indexy, included sloupce
  • query filters – HasQueryFilter
  • konverze hodnot – HasConversion
  • mapování na view – ToView, JSON – ToJson

Validační atributy – schéma NEmění

[Range][EmailAddress][Phone][Url][RegularExpression][Compare][DataType][Display][MinLength]

Používá je validace v ASP.NET Core / Blazor – EF Core je při tvorbě schématu ignoruje.

Výjimky, které schéma mění: [Required] (NOT NULL) a [MaxLength] / [StringLength] (délka).
Závěr

Shrnutí

  • Tři vrstvy konfigurace: konvence → anotace → Fluent API (silnější přebíjí slabší)
  • Atributy bydlí ve třech namespacech – EF-specifické v Microsoft.EntityFrameworkCore
  • Složený primární klíč: [PrimaryKey] na třídě (EF 7+) – ne dvakrát [Key]
  • [Index] patří na třídu – unikátní, složené, pojmenované, řazené; stejná syntaxe v EF 9 i EF 10
  • Validační atributy schéma nemění – [Range], [EmailAddress]… slouží jen ASP.NET validaci
  • Kompletní přehled všech atributů s příklady → referenční seznam (samostatná HTML stránka)
